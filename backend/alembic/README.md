# Alembic migrations

Autogenerate a new revision after changing models:

```
make migrate-new msg="add foo column"
```

Apply pending migrations:

```
make migrate
```

Rollback the most recent migration:

```
make migrate-down
```

All migration files live in `alembic/versions/`. Review the autogenerated
diff before committing — Alembic doesn't always get index changes or enum
edits right.
